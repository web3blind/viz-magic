### Баги из чата с тестировщиком и что сделано

#### 1. Приглашения в гильдию и список гильдий не видны на твинке
- **Причина:** Окно синхронизации было 1000 блоков (~50 мин). Свежий клиент не видел гильдии старше часа.
- **Фикс (app.js):** Окно увеличено до 28800 блоков (~24ч). Батч при догонке — 50 вместо 10.
- **Коммит:** `728c52f`

#### 2. Основатель не может покинуть гильдию
- **Причина:** Движок запрещает (by design), но UI показывал кнопку без обратной связи.
- **Фикс (guild.js):** Кнопка скрыта для основателя, вместо неё — текст `guild_founder_cannot_leave`.
- **Не реализовано:** Передача лидерства другому участнику (нет action type для этого).
- **Коммит:** `728c52f`

#### 3. Вызов на дуэль не виден оппоненту
- **Причина:** Arena сканировала только 100 блоков (~5 мин) для входящих вызовов.
- **Фикс (arena.js):** Увеличено до 2400 блоков (~2ч). Основной фикс синхронизации (п.1) тоже помогает.
- **Коммит:** `728c52f`

#### 4. Неправильные параметры противника в дуэли
- **Причина:** До гидратации с блокчейна показывается заглушка (уровень 1). `_ensureCharacterHydrated` работает асинхронно — после загрузки экран перерисовывается. Основной фикс синхронизации решает проблему.
- **Коммит:** `728c52f`

#### 5. Результаты мирового босса не видны другим
- **Причина:** UI вызывал `WorldBoss.attackBoss()` напрямую (локально), без записи в блокчейн.
- **Фикс (world-boss.js):** Атака теперь идёт через `VizBroadcast.gameAction()` с оптимистичным локальным обновлением.
- **Коммит:** `728c52f`

#### 6. Баг расчёта урона босса
- **Причина:** Код обращался к `character.stats.pot`, но у персонажей нет подобъекта `.stats` — stat `pot` лежит на верхнем уровне.
- **Фикс (state-engine.js, world-boss.js):** Используется `CharacterSystem.getTotalStat(character, 'pot')`.
- **Коммит:** `728c52f`

#### 7. Гильдии старше 24ч невидимы для свежих клиентов
- **Решение:** Новый action type `guild.listing`. Офицеры жмут "Продвинуть гильдию" → в блокчейн записывается `{guild_id, created_block}`. Свежие клиенты подбирают листинги, запрашивают блок создания, гидратируют полные данные гильдии.
- **Файлы:** config.js, guild-protocol.js, state-engine.js, guild.js, i18n
- **Подсказка в UI:** "Продвигайте гильдию раз в сутки, чтобы она была видна новым игрокам."
- **Коммит:** `ae5ffac`

#### 8. Справка, README и подсказки не соответствовали коду
- Добавлены 3 раздела справки: Дуэли/Арена, Гильдии, Мировой Босс
- `boss_attack_cost` говорил "Стоит 200 Маны" — код не списывает ману. Заменено на "Урон зависит от Мощи и уровня".
- HP regen в README: было "500 блоков ~4 мин", исправлено на "100 блоков ~5 мин" (по config.js).
- Убраны устаревшие VIZ-ноды из README (solox, lexa).
- Обновлено описание гильдий в README.
- **Коммит:** `015c93a`

#### 9. Стоимость атаки босса — 2% маны
- Атака босса теперь стоит 200 bp (2% маны) через award к аккаунту босса.
- Проверка энергии перед атакой, сообщение об ошибке если маны не хватает.
- **Коммит:** (текущий)

#### 10. Ускорение начальной синхронизации
- Батч увеличен до 200 блоков при отставании >1000.
- Пустые блоки (без VM/V операций) обрабатываются без задержки при догонке.
- Блоки с контентом — задержка 10мс вместо 50мс при догонке.
- **Коммит:** (текущий)

### Известные нерешённые вопросы

1. **Передача лидерства гильдии** — основатель не может уйти, но нет action type для передачи роли founder другому участнику.
2. **Раздача готовых checkpoint-файлов** — для совсем новых клиентов можно генерировать и хостить готовый checkpoint JSON, чтобы пропустить синхронизацию целиком.

### Архитектурные заметки
- ES5 strict, IIFE модули, без фреймворков и сборки
- Все действия идут через `VizBroadcast.gameAction()` → блокчейн → `BlockProcessor` → `StateEngine`
- Оптимистичные обновления: UI обновляется сразу, блокчейн подтверждает через ~3 сек
- Стейт сохраняется в IndexedDB через `CheckpointSystem`
- i18n: все строки в `app/js/i18n/ru.js` и `en.js`, доступ через `Helpers.t('key')`
